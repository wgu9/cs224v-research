
# Appendix

### 表1：智能体漂移现象的分类与定义

下表从更宏观的“智能体漂移”（Agent Drift）视角出发，对不同类型的漂移进行了分类和定义，以厘清“上下文漂移”在整个问题域中的位置 [1]。

| 漂移类型 | 定义 | 主要特征 |
| --- | --- | --- |
| **目标漂移 (Goal Drift)** | 智能体接收到的任务目标分布或其语义解释随时间发生变化 [1]。 | 输入驱动；通常是漂移因果链的起点，可能引发后续的上下文或推理漂移 [1]。 |
| **上下文漂移 (Context Drift)** | 智能体用于决策的活动上下文（如检索的文档、对话历史）中，相关信息的丢失、损坏或质量下降 [2, 1]。广义上指模型输出与目标一致性行为之间逐渐产生的时序性偏离 [3, 2]。 | 状态驱动；表现为一种时序性的、缓慢的“意图侵蚀” [2]，而非瞬时发生的故障。 |
| **推理漂移 (Reasoning Drift)** | 智能体核心的规划、逻辑推理或决策生成能力出现下降 [1]。 | 认知能力驱动；可能由上下文漂移或模型自身退化引起，并导致协作漂移 [1]。 |
| **协作漂移 (Collaboration Drift)** | 智能体与外部工具、API或其他智能体交互的性能下降 [1]。 | 交互驱动；通常是推理漂移在执行层面的最终体现，如工具调用失败 [1]。 |
| **对齐漂移 (Alignment Drift)** | 智能体的行为偏离了预设的行为策略、安全准则或伦理价值观 [2]。 | 行为策略驱动；与任务能否成功不直接相关，但可能与上下文漂移同时发生 [2]。 |

### 表2：上下文漂移的具体评估维度

下表聚焦于“上下文漂移”本身，根据您的具体要求，将其分解为可操作、可评估的几个核心维度。

| 维度 | 定义 | 典型表现 |
| --- | --- | --- |
| **范围不一致 (Scope Incoherence)** | 智能体执行了超出任务明确或隐含边界的操作。 | 在软件工程任务中修改了不相关的文件；在网页导航时访问了与任务无关的网站；超出了用户的核心意图范围。 |
| **工具误用 (Tool Misuse)** | 智能体以不符合任务目标的方式使用工具或调用API。 | 选择了不相关的工具；为API调用提供了错误的参数；以冗余、低效或不合逻辑的顺序调用工具。 |
| **错误固化 (Error Persistence)** | 智能体在收到失败反馈后，仍然重复执行相同的错误动作，或陷入无进展的行为循环。 | 在Web Arena中反复点击一个无效的按钮；在编码任务中反复提交无法通过相同单元测试的代码；陷入“失忆式执行”，无法从过去的错误中学习 [4]。 |
| **决策缺乏依据 (Justification Failure)** | 智能体的行为无法由当前可用的证据或其自身的推理链条来支撑。 | 幻觉出环境的状态或不存在的事实 [5]；无法为其决策提供合理的解释或理由。 |
